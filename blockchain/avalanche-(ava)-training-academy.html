What is consensus?

Important

Consensus plays a crucial role in blockchain networks by resolving conflicts and ensuring that all validators agree on the current state of the distributed ledger.

Main goal:

  • create a single version of truth that is universally accepted by network participants.

Consensus protocol:

  • set of steps followed by validators to collectively decide on the state of the system and all state changes

Protocols in Avalanche family operate through repeated sub-sampled voting. Majority is how consensus is achieved

Consecutive Successes

  • Avalanche consensus does not run for number of fixed rounds, but until a threshold is hit.
  • sampling continues until 8 rounds is hit

Parameters of Avalanche Consensus

Symbol Name Range Explanation
n Number of Participants 1 to ∞ How many participants take part in the system?
k Sample Size 1 to n How many of these participants get asked every round of sub-sampiling?
α Quorum Size 1 to k How many of the asked participants have to have the same preference for me to change my preference?
β Decision Threshold >= 1 How many times does the quorum have to confirm my preference until I finalize my decision?

Double Spending Attack

Important

Double spending attack is when a user attempts to spend more crypto than they own by creating multiple transactions that reference the same funds

Essentially, consensus is when a validator either confirms or denies a transaction based upon which transaction has an earlier timestamp.

  • transaction with the earlier timestamp will be written to the block
  • this eliminates the double spending vulnerability

Guarantee of high probability:

Important

If any honest validator accepts a transaction, all honest validators will accept the same transaction.

Time to finality > time from beginning transaction to end
Throughput > the number of lines the highway has, so to speak

  • avalanche has a short highway with many lanes, making it very efficient in transaction speed, volume, and scalability
    • compared to Ethereum and Bitcoin, which have less efficient throughput, with the latter being the slowest

Finalization

  • when a transaction has no conflicts, finalization happens quickly
  • when conflicts exist, honest validators quickly cluster around conflicting transactions, entering a positive feedback loop until all correct validators prefer that transaction.
