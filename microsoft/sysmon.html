Downloading Sysmon

Go to Microsoft's website and download the zipped file
https://learn.microsoft.com/en-us/sysinternals/downloads/sysmon

Extract the files

Well balanced configuration file for logging Sysmon Events
https://github.com/olafhartong/sysmon-modular/tree/master

Alternatively, the Swift on Security could be installed. The link to that configuration is here: https://github.com/SwiftOnSecurity/sysmon-config

Whichever you choose, go to either sysmonconfig.xml or sysmonconfig-export.xml and download the raw file.

Move the XML file into the folder with the Sysmon executable

If successful, you should see an output like this:

Now, we can verify this in Event Viewer.

While in your PowerShell prompt, open the calculator app.

calc.exe

Open Event Viewer

Scroll down until you see Sysmon

Click on Operational and you should see some events

If we filter for Event ID 10 (Process Access), we can see a couple events

Looking at the log a little bit closer, we see a few things

  • Source Image C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe - remember we were in the PowerShell prompt and typed calc.exe? This is basically saying the application that started the calculator app, was powershell.exe. The source image is a very crucial piece of information in investigating a user's activities.
  • Target Image C:\Windows\system32\calc.exe - This is the target process that is being accessed in this particular event. Knowing the target image can help in identifying and unintended or malicious processes that may have been executed.

Right after Event ID 10, we can see an Event ID 7 (Image Loaded) for calc.exe

That's just a very basic run through of the capabilities in Event Viewer. It can be very helpful to know how to navigate the different types of logs and log sources.

Windows Event Logs

To see more information about Windows Event Logs, check out my other page Windows Event Logs, Windows System Logs, and a great guide to Windows Forensics, Windows Forensics Handbook.

Sysmon Event ID's

Event Name Event ID Description Relation to Cybersecurity
Process Creation 1 Logs details of process creation, including command-line arguments. Useful for detecting suspicious processes, malware execution, or command-line abuse (e.g., PowerShell misuse).
File Creation Time Changed 2 Records when a file creation timestamp is changed. Can detect timestamp manipulation (e.g., attackers trying to hide their tracks by altering file timestamps).
Network Connections 3 Logs TCP/UDP network connections established by a process. Useful for monitoring outbound connections and detecting C2 (Command & Control) traffic from malware.
Sysmon Service State Change 4 Tracks changes in Sysmon service state (start/stop). Detects tampering with Sysmon logging, which may indicate adversaries disabling monitoring to avoid detection.
Process Termination 5 Logs process termination. Can help in tracking process lifecycles to identify abnormal or suspicious process terminations.
Driver Load 6 Logs when a driver is loaded. Useful for detecting unauthorized or malicious driver loading, often associated with rootkits or persistence.
Image Load 7 Records the loading of modules (DLLs) into processes. Detects suspicious DLL injection or loading of malicious libraries by compromised processes.
CreateRemoteThread 8 Logs the creation of threads in a remote process. Can detect process injection techniques used by malware to execute code in the context of another process.
Raw Access Read 9 Logs raw disk access by a process. Detects attempts to bypass OS protections or tampering with file systems, often used in forensic evasion.
Process Access 10 Logs access to another process's memory. Useful for detecting privilege escalation or lateral movement techniques, such as token theft or injection.
File Creation 11 Logs file creation events. Helps detect suspicious or malicious file creation activities, such as malware writing to disk.
Registry Object Added/Deleted 12 Logs creation or deletion of registry keys or values. Useful for monitoring registry changes related to persistence mechanisms or malware configuration changes.
Registry Value Set 13 Logs setting of a registry value. Detects registry modifications, such as changes to startup entries or security-sensitive registry keys.
Registry Object Renamed 14 Logs when a registry key or value is renamed. Detects attempts to rename registry objects to evade detection or establish persistence.
FileStream Creation 15 Logs the creation of alternate data streams (ADS) on files. Alternate data streams are often used by malware for hiding data or maintaining stealth.
File Rename 16 Logs file rename events. Can help detect suspicious file renaming operations, which could indicate attempts to hide malicious files.
DNS Query 22 Logs DNS queries made by a process. Useful for detecting domain names related to malware C2 or exfiltration domains, as well as DNS tunneling.
File Delete 23 Logs file deletion events. Helps track file deletions, which could be part of data destruction or cleanup activities by malware.
Clipboard Change 24 Logs changes to clipboard content. Can help detect exfiltration of sensitive data or clipboard-based attacks like credential harvesting.
