Overview of Risk Reports

Microsoft Entra ID Protection offers several reports to investigate identity risks:

  1. Risky users
  2. Risky sign-ins
  3. Risky workload identities
  4. Risk detections
Important

These reports can be accessed through the Microsoft Entra admin center under Protection > Identity Protection. They allow for data export in CSV or JSON formats and integration with SIEM tools.

Types of Risk Reports

Risky Users Report

This report shows users whose accounts are or were at risk of compromise. Key features include:

  • Filters and customizable columns
  • Detailed user information, including risk level and history
  • Options to reset passwords, confirm compromise, mark users safe, or dismiss risks
  • Integration with Microsoft 365 Defender for further investigation

Risky Sign-ins Report

This report covers both interactive and non-interactive sign-ins for up to 30 days, providing:

  • Risk levels and detection types
  • Conditional Access policy information
  • MFA, device, and location details
  • Options to confirm sign-ins as compromised, safe, or dismiss risks

Risk Detections Report

Covering up to 90 days of data, this report offers:

  • Detailed information on each risk detection
  • MITRE ATT&CK framework categorization
  • Contextual information about simultaneous risks and sign-in locations

Investigation Process

The article recommends a structured approach to investigating risks:

  1. Initial triage using the ID Protection dashboard and Impact analysis workbook
  2. Configuring named locations and known traveler databases
  3. Reviewing logs for patterns and potential compromises
  4. Enabling self-remediation policies

Investigation and Risk Remediation Framework

A flowchart is provided, outlining steps from risk detection to resolution:

  1. Attempt self-remediation
  2. Administrator-led remediation if necessary
  3. Further investigation to determine if the risk is safe, compromised, or should be dismissed

Specific Detection Investigations

The article provides detailed guidance on investigating various types of detections:

  • Microsoft Entra threat intelligence
  • Atypical travel
  • Anomalous token and token issuer anomalies
  • Suspicious browser detections
  • Malicious IP address detections
  • Password spray detections
  • Leaked credentials
    For each type, the article offers recommended actions based on different scenarios, such as confirming compromises, marking sign-ins as safe, or implementing additional security measures.

Key Takeaways

  1. Microsoft Entra ID Protection offers comprehensive tools for risk investigation and mitigation.
  2. A structured approach to risk investigation is crucial for effective security management.
  3. Different types of risk detections require specific investigation strategies and remediation actions.
  4. Integration with other Microsoft security tools enhances the investigation process.
  5. Continuous monitoring and prompt action on detected risks are essential for maintaining security.
