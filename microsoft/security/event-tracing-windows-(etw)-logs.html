Event Tracing Windows (ETW) Logs

ETW is a powerful tracing facility built into Windows, primarily used for performance monitoring, diagnostics, and troubleshooting. It can be leveraged for security detection because it provides detailed, low-level logging across various system activities.

Below is a table that aligns ETW events with similar attack types listed in Windows Event Logs

Table: ETW Logs Correlated with Common Attack Types

Attack Type Relevant ETW Providers & Event Fields ETW Description How It Relates to Cybersecurity
User Logon/Authentication Microsoft-Windows-Security-Auditing (Event IDs: 4624, 4625) Tracks user logon and logoff events, including failed login attempts. Provides detailed information on logon failures or suspicious logon times, useful for detecting brute force attacks or compromised credentials.
Microsoft-Windows-Kernel-Process (Event ID: 1) Logs process creation with command-line arguments. Useful for detecting malicious tools or scripts used post-compromise, especially during the lateral movement stage.
Privilege Escalation Microsoft-Windows-Security-Auditing (Event ID: 4672, 4720) Logs privilege assignments and user account creations. Monitors critical security changes where attackers might escalate privileges after gaining initial access.
Microsoft-Windows-Kernel-Audit-API-Calls (Event IDs: various) Logs system calls related to access token manipulation. Attackers often abuse access tokens (e.g., stealing or impersonating tokens), making this valuable for detecting privilege escalation activities like Pass-the-Token attacks.
File and Folder Access Microsoft-Windows-Security-Auditing (Event ID: 4663) Tracks file and object access, including read, write, and delete actions. Monitors sensitive file access, helping to detect ransomware or exfiltration attempts. Unusual access to system files could indicate a malware attack.
Microsoft-Windows-Kernel-File (Event ID: 11) Tracks file creation, modification, and deletion. Used to monitor suspicious file system activity that might indicate malware actions, such as file creation in sensitive directories (e.g., %TEMP% or %APPDATA%).
Microsoft-Windows-Sysmon (Event ID: 11 - FileCreate) Logs when a file is created. Sysmon ETW logs provide more granular visibility into file creation, especially helpful for tracking new executables created by malware or during data staging for exfiltration.
Process Creation Microsoft-Windows-Kernel-Process (Event ID: 1) Logs all process creations, including command-line parameters and parent processes. Detects malware spawning child processes or command-line execution of suspicious processes (e.g., PowerShell or CMD.exe commands seen in post-exploitation).
Network Activity Microsoft-Windows-TCPIP (Event ID: 10, 11) Logs network connections (both inbound and outbound), including details like source and destination IP addresses. Essential for tracking suspicious outbound connections to external servers, which could indicate data exfiltration or communication with Command and Control (C2) servers.
Microsoft-Windows-Sysmon (Event ID: 3 - NetworkConnect) Logs network connections with detailed information such as process ID, IP, and port. Valuable for identifying connections from suspicious processes to unfamiliar IP addresses. This can help in detecting C2 communications in attacks like Malware or RATs.
Persistence Techniques Microsoft-Windows-Kernel-Process (Event ID: 1) Logs the creation of new processes, including service processes. Monitoring for unusual processes (like newly created scheduled tasks or services) helps detect persistence techniques used by attackers to maintain access.
Microsoft-Windows-Sysmon (Event ID: 1 - ProcessCreate) Logs new processes, including parent processes and full command-line arguments. Sysmon adds richer context, such as parent-child process relationships, helping to detect suspicious or hidden tasks that could be related to persistence.
Audit and System Tampering Microsoft-Windows-Security-Auditing (Event ID: 4719, 1102) Tracks changes in audit policies and log clearing actions. Identifies potential tampering when attackers disable auditing to hide their tracks, or clear security event logs post-compromise.
Microsoft-Windows-Kernel-Audit-API-Calls (Event IDs: various) Logs low-level system calls to modify processes or change audit configurations. Useful for detecting unauthorized tampering with security configurations, possibly by malware or advanced threats aiming to disrupt monitoring.
Active Directory Attacks Microsoft-Windows-Security-Auditing (Event ID: 4932) Logs replication events of Active Directory objects. Detects potential replication of AD data, which might indicate attempts to steal credentials or compromise sensitive directory objects (e.g., DCSync attack).
Microsoft-Windows-Security-Auditing (Event ID: 4673) Logs attempts to perform privileged operations. Useful for detecting unauthorized attempts to perform AD-related attacks or escalate privileges in an AD environment (e.g., Kerberoasting).

Key ETW Providers for Security Monitoring

  1. Microsoft-Windows-Security-Auditing: This is one of the most useful providers when tracking security events, including user authentication, privilege escalation, and object access.
  2. Microsoft-Windows-Kernel-Process: Logs crucial details about process creation and termination. Attackers often spawn new processes during post-exploitation.
  3. Microsoft-Windows-Sysmon: Provides enhanced visibility into process creations, network connections, file access, and other suspicious behavior. Sysmon is especially valuable when conducting advanced attack detection and forensic investigations.
  4. Microsoft-Windows-TCPIP: Tracks TCP/IP connections, useful for identifying suspicious or unauthorized network traffic.
  5. Microsoft-Windows-Kernel-File: Monitors file system activity, which helps in detecting file-based attacks like ransomware or data staging for exfiltration.

Practical Use in Attack Detection

  • Brute Force & Credential Theft: Use ETW providers like Security-Auditing to correlate Failed Logons (4625) with other logon attempts to detect brute force or stolen credential attempts.
  • Privilege Escalation & Persistence: Monitor Process Creation (Kernel-Process) and Sysmon ProcessCreate (1) logs to detect unusual processes, especially those launched by administrators or SYSTEM accounts.
  • Network-based Attacks: Combine Sysmon’s NetworkConnect (3) and TCPIP Event IDs to track network behavior, focusing on external connections to unknown IP addresses, which could indicate exfiltration or C2 communication.

Summary

ETW logs provide detailed system-level visibility and, when combined with traditional Windows Event Logs, can significantly enhance detection capabilities against cyber threats. By leveraging ETW, you get a more granular view of what’s happening on the system, from process creation to network connections, making it harder for attackers to hide their tracks.
