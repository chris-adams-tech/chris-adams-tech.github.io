Vulnerable AD Lab

Basic Installation Steps:

  1. Install Windows Server 2022 on a virtual machine
  2. Install Ubuntu Server on same subnet on a virtual machine (not in these steps)
  3. Patch servers
  4. Install Active Directory w/ Domain Services and promote server to domain controller
  5. Install Sysmon with Swift on Security configuration
  6. Install Wazuh Manager/Server on Ubuntu via Quick start guide
    1. https://documentation.wazuh.com/current/quickstart.html
  7. Once Sysmon is installed, deploy script via Powershell on domain controller
  8. Download & run vulnerable Powershell script
  9. Set DNS to internal, remove default gateway

2. Setting up the Active Directory server

Open Powershell as administrator:

# To add Domain Services role to Server Manager
PS C:\Users\Administrator> 
Install-WindowsFeature -Name ad-domain-services -IncludeManagementTools

Then run

# To install Active Directory, run this script. Change the domain to whatever you prefer
PS C:\Users\Administrator> 
Install-ADDSForest -CreateDnsDelegation:$false -DatabasePath "C:\\Windows\\NTDS" -DomainMode "7" -DomainName "insert.domain" -DomainNetbiosName "domain" -ForestMode "7" -InstallDns:$true -LogPath "C:\\Windows\\NTDS" -NoRebootOnCompletion:$false -SysvolPath "C:\\Windows\\SYSVOL" -Force:$true

Next logon a login screen with the domain created should appear

    Extract packages from Windows download

    Important

  2. Open a Powershell window with admin privs and move to the Downloads\Sysmon folder, then run the command below:

    PS C:\Users\*****\Downloads\Sysmon> .\sysmon64.exe -accepteula -i sysmonconfig-export.xml

    If adding the SoS sysmon config, be sure the file is in the unextracted Sysmon folder
