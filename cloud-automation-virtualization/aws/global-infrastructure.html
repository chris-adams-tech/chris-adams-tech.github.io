Global Infrastructure

By creating a global application, this allows access from anywhere in the world. Of course, knowing the users that will be accessing the resource will help to determine where to place to application.

Having your application hosted globally can have the following benefits:

  • Decreased Latency
  • Disaster Recovery
  • Attack Protection

Global Applications of AWS

Global DNS: Route 53 - routes users to closest deployment with least latency

  • managed DNS
  • great for disaster recovery strategies
• In AWS, the most common records are:
• www.google.com => 12.34.56.78 == A record (IPv4)
• www.google.com => 2001:0db8:85a3:0000:0000:8a2e:0370:7334 == AAAA IPv6
• search.google.com => www.google.com == CNAME: hostname to hostname
• example.com => AWS resource == Alias (ex: ELB, CloudFront, S3, RDS, etc…)

Route 53 Routing Policies

  • Simple Routing - no health checks
  • Weighted Routing - essentially load balances the EC2 instances
    • this implements health checks

Global Content Delivery Network** (CDN) CloudFront

  • replicate part of your application to AWS Edge locations - decrease latency
  • improves read performance
    • content cached at the edge
  • 216 Points of Presence globally
  • DDoS protection

*S3 Transfer Acceleration** - accelerate global uploads & downloads into Amazon S3

CloudFront Origins

S3 bucket
  • for distributing files and caching them at the edge
  • enhanced security with CloudFront Origin Access Control (OAC)
  • OAC is replacing Origin Access Identity (OAI)
  • CloudFront can be used as an ingress
Custom Origin (HTTP)
  • Application Load Balancer
  • EC2 instance
  • S3 website
  • Any HTTP backend you want
CloudFront vs. S3 Cross Region Replication

CloudFront

  • global Edge network
  • files are cached for TTL (maybe a day)
  • Great for static content that must be available everywhere

S3 Cross Region Replication

  • must be setup for each region you want replication to happen
  • files are updated near real-time
  • read only
  • Great for dynamic content that needs to be available at low-latency in few regions

S3 Transfer Acceleration

  • enables fast, easy, and secure transfers of files over long distances between your client and an S3 bucket.

AWS Global Accelerator

  • improve global application availability and performance using AWS global network
  • 2 Anycast IP are created for application and traffic is sent through this

AWS Outposts

AWS Outposts are "server racks" and offer the same infrastructure, services, APIs and tools to build your own applications on-premises.

In this case, you are the one responsible for the physical security of the server rack.

AWS WaveLength

Infrastructure deployments embedded within the telecommunications providers datacenters at the edge of 5G networks.
