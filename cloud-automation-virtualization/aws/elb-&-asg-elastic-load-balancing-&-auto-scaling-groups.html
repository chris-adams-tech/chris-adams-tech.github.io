Scalability = applications and system can handle greater loads by adapting

Types of Scalability
  • Vertical scalability
    • increasing size of instance
    • example: going from t2.micro to t2.large
    • common for non-distributed systems, such as databases
    • usually a limit (hardware limit)
  • Horizontal scalability (elasticity)
    • increasing number of instances
    • implies distributed systems
    • common for web/modern applications
High Availability

High availability typically works alongside horizontal scaling

  • Means running your application in at least 2 Availability Zones
  • goal is to survive data center loss
  • creates multiple locations

Elastic Load Balancing (ELB)

Load balancers are servers that forward internet traffic to multiple servers (EC2 Instances) downstream.

  • Load balancers help to spread data load across multiple downstream instances
  • Can expose single point of access (DNS) to your app
  • Handles failures of downstream instances
  • Do regular health checks to your instances
  • Provide SSL termination (HTTPS) for your websites
  • HA across zones

AWS is managed and is provisioned by AWS.

  • AWS maintains upgrades, maintenance, high availability
  • provides minimal configuration
4 Types of Load Balancers from AWS
  1. Application Load Balancer - HTTP/HTTPS - Layer 7
  2. Network Load Balancer - ultra-high performance, allows TCP - Layer 4
  3. Gateway Load Balancer - Layer 3
  4. Classic Load Balancer (retired 2023) - Layer 4 & 7

Auto Scaling Groups (ASG)

Scaling Strategies

Manual Scaling - update size of ASG manually

Dynamic Scaling - respond to changing demand

  • Simple / Step scaling
    • When CloudWatch alarm is triggered (example: CPU > 70%, then add 2 units
    • When CloudWatch alarm is triggered (example: CPU < 30%), then remove 1 unit
  • Target Tracking Scaling
    • Example: ASG CPU to stay around 40%
  • Scheduled Scaling
    • anticipate scaling based on known usage patterns

**Predictive Scaling

  • Uses machine earning to predict future traffic ahead of time
  • Automatically provisions the right number of EC2 instances
  • Use when there are predictive time-based patterns
