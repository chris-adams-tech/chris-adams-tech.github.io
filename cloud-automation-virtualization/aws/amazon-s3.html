Amazon S3

S3 is one of the main building blocks of AWS

Use Cases
  • Backup and storage
  • Disaster Recovery
  • Archive
  • Hybrid Cloud Storage
  • Application hosting
  • Media hosting
  • Data lakes & Big data analytics
  • Software delivery
  • Static Website

Buckets

Files in S3 = Objects

Directories in S3 = Buckets (must have globally unique name across all regions and accounts, though they are defined at the region level)

  • S3 buckets are globally accessible but must be defined in one region.
Naming convention:
  • No uppercase, No underscore
  • 3-63 characters long
  • Not an IP
  • Must start with lowercase letter or number
  • Must NOT start with prefix xn--
  • Must NOT end with the suffix -s3alias

S3 Objects

  • Objects (files) have a key
  • key is the FULL path:
    • s3://my-bucket/my_file.txt
    • s3://my-bucket/my_folder/another_folder/my_file.txt
  • Key is compose of prefix + *object name
    • s3://my-bucket/my_folder/another_folder/my_file.txt
  • No concept of directories within buckets

Maximum object size is 5TB (5000 GB)

  • if larger than 5 GB, must use "multi-part upload"

Security

User-based

IAM Policies

IAM Policies based around which API calls should be allowed for a specific user from IAM

Resource Based

  • Bucket policies - bucket wide rules from S3 console -- allows cross account
  • Object Access Control List (ACL) - finer grain (can be disabled)
  • Bucket Access Control List (ACL) - less common (can be disabled)
Note

IAM principal can access an S3 object if

  • user IAM permissions ALLOW it OR the resource policy ALLOWS it AND there's not explicit DENY
Encryption

Objects are encrypted using Amazon S3 encryption keys

S3 Bucket Policies

  • JSON based policies
    • Resources: buckets and objects
    • Actions: Set of API to Allow or Deny
    • Principal: - account or user to apply the policy to
  • Use cases:
    • grant public access to bucket
    • Force objects to be encrypted at upload
    • grant access to another account

S3 Versioning

  • Versioning is enabled at the bucket level
  • same key will overwrite will change the version number
  • best practice to version your buckets
    • protects against unintended deletes
    • easy roll back to previous versions

S3 Storage Classes

Amazon S3

  • Standard - General Purpose
    • 99.99& availability
    • used for frequently accessed data
    • low latency, high throughput
    • sustain 2 concurrent facility failures
    • Use Cases: big data analytics, mobile & gaming applications, content distribution
  • Standard-Infrequent Access (IA)
    • data less frequently accessed, but requires access when needed
    • lower cost than S3 standard
  • One Zone-Infrequent Access
  • Glacier Instant Retrieval
  • Glacier Flexible Retrieval
  • Glacier Deep Archive
  • Intelligent Tiering

S3 Durability and Availability

Durability

If you store 10,000,000 objects within Amazon S3, you can on average expect to incur a loss of a single object once every 10,000 years

Availability

  • measures how readily available a service is
  • varies depending on storage class
  • S3 standard has 99.99% availability
    • this means it's not available 53 minutes per year

S3 Encryption

AWS provides Server-side encryption by default.

  • Server Side Encryption - server encrypts the file after receiving it
  • Client-side Encryption - client encrypts the file before uploading it

IAM Access Analyzer for S3

  • Ensures only intended people have access to your S3 buckets
  • Evaluates S3 bucket policies, S3 ACLs, S3 Access Point Policies

AWS Snow Family

*Highly secure, portable devices to collect and process data at edge, and migrate data into and out of AWS.

  • These are offline hardware devices to perform data migrations
  • Devices get shipped to AWS for upload to S3.

Usage Process

  1. Request Snowball device from AWS console
  2. Install Snowball client / AWS OpsHub on servers
  3. Connect Snowball to your servers and copy files using the client
  4. Ship back the device
  5. Data will be uploaded into S3 bucket
  6. Snowball is completely wiped

Edge Computing

Process data while it's being created on an edge location

  • truck on road, ship on sea, mining station
  • may have limited network access and no CPU power

Snowball Edge/Snowcon device can be used:

  • *Snowcone: 2 CPUs, 4 GB memory, wired or wireless access
  • Snowball Edge Compute Optimized & Storage Optimized
  • Run EC2 Instances or Lambda functions at the edge

Use Cases: preprocess data, machine learning, transcoding media
