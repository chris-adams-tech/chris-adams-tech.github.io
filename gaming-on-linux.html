Gaming on Linux

Finally some more movement on Steam - Linux integration!

My journey into the Linux environment began from a wanting to break free from Windows environments that run so many unnecessary processes. Since I was so comfortable with Windows, (i.e. the navigation, file system, daily operations) it was rather tough to make a decision to change. Additionally, it was tough because there was a host of gaming platforms that initially was near impossible to run Steam and Windows systems that use DOS to boot.

Then, I came to find out that gaming on Linux has started to become more popular. So...the research began...

I began using WSL, or Windows Subset for Linux, which allows using a bash shell in Windows. I started off with that. I thought, okay, this is not too tough. Once you have the basics down, it's pretty easy to navigate, right?

Then, after researching further into Linux distributions, I found a Ubuntu based system called Linux Mint. Here's their website: https://linuxmint.com/
I then learned, that even if you don't fully know command line that you could run this system, for free, and could still have it set up similar to what a Windows environment looks like.

Though I never thought it before, I realized how much the aesthetic of a desktop can help my workflow. This may not be the case for everyone, but when I have a functional workspace, that is optimized for navigation with the software that is installed, it becomes a whole different productive experience.

I was sold, so naturally, to test it out, I first implemented the system on a virtual machine using VMware. At this point, I didn't really have much virtualization experience. For my first experience, I kind of felt like I was home. Everything ran so fast, I was able to navigate similar to Windows, and even have additional customization options.

This was cool, but at this time, it wasn't able to fully boot Steam while running the NVIDIA graphics card. My research continued...

Then, I found an Arch distribution called Hyprland, and this OS was beauuuutiful. Though, it came with a price, in that, it was extremely tough to configure and maintain. That's the thing about some Linux distributions that I've come to learn. Everything doesn't just work out of the box, there are packages, dependencies, bit rate file systems, lvm containers, etc. So, it's just different. You learn what works for you and how to troubleshoot it. For me, it was an incredible learning experience as I just sort of dove in head first.

I'm a figure it out kind of guy and this really tested my patience of troubleshooting, as I didn't have nearly the experience I should have when booting this system. I was determined though, I wanted to run Steam on Linux.

After a lot of trials and tribulations, I had finally got it booted and was running Steam on Linux with the native NVIDIA graphics card. With this, I was also able to run Windows games such as The Witcher 3 and Cyberpunk 2077.

A cool new fix they have been using is a wrapper, Proton that essentially allows booting of Windows DOS applications. So now, I can boot any Windows application into Linux, no problem.

All in all, I think in today's age privacy is extremely important, which is what led me to using Linux systems. I am a advocate for privacy over convenience.

I regularly keep up with Linux gaming news and had recently seen this post,
https://www.gamingonlinux.com/2024/10/valve-makes-a-big-improvement-for-native-linux-games-in-a-steam-beta-update/ and is really great to see so much progress in the Linux gaming world. Though, I have not been gaming as much because of lack of time, I think it will be something I will always keep up, as I love seeing the innovation and expansion.
