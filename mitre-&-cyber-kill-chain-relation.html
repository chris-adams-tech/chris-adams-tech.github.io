MITRE & Cyber Kill Chain Relation

Relation to MITRE ATT&CK Techniques

The MITRE ATT&CK framework takes a much more granular approach compared to the broad stages of the Cyber Kill Chain. Instead of focusing just on the stages of the attack lifecycle, ATT&CK provides a matrix of specific tactics (the what) and techniques (the how) that adversaries use to achieve their goals.

Tactics: These represent the adversary's goals (e.g., "Initial Access", "Privilege Escalation").
Techniques: These are the methods adversaries use to accomplish each tactic (e.g., using spear-phishing to gain initial access).

Comparing Cyber Kill Chain and ATT&CK

Both frameworks share common ground, as they describe similar activities an attacker might perform. However, MITRE ATT&CK breaks down each stage of the Cyber Kill Chain into more detailed steps, giving defenders more specific, actionable insights.

Here’s how each phase of the Cyber Kill Chain aligns with MITRE ATT&CK tactics:

Reconnaissance (Cyber Kill Chain) ↔ Reconnaissance (ATT&CK):

Cyber Kill Chain's reconnaissance maps to ATT&CK’s Initial Access and Reconnaissance tactic.

ID Technique
T1566 Phishing
T1598 OSINT gathering

Weaponization (Cyber Kill Chain) ↔ Resource Development (ATT&CK):

The creation of malware or exploits corresponds to ATT&CK’s Resource Development tactic

ID Technique
T1587 Malware Development
T1588 Code Signing Certificates

Delivery (Cyber Kill Chain) ↔ Initial Access (ATT&CK):

The method by which the attacker gains access corresponds to ATT&CK’s Initial Access tactic. Common techniques include Spearphishing Attachment (T1566.001) or Drive-by Compromise (T1189).

ID Technique
T1566.01 Spearphishing
T1189 Drive-by Compromise

Exploitation (Cyber Kill Chain) ↔ Execution/Privilege Escalation (ATT&CK):

Exploiting vulnerabilities or running malicious code maps to ATT&CK’s Execution and Privilege Escalation tactics.

ID Technique
T1068 Exploitation for Privilege Escalation

Installation (Cyber Kill Chain) ↔ Persistence (ATT&CK):

Installing malware or a backdoor corresponds to ATT&CK’s Persistence tactic. Techniques like Create or Modify System Process (T1543) could be used to maintain long-term access.

ID Technique
T1543 Create of Modify System Process

Command and Control (Cyber Kill Chain) ↔ Command and Control (ATT&CK):

The attacker’s remote control of the compromised system is covered under ATT&CK’s Command and Control tactic.

ID Technique
T1105 Ingress Tool Transfer
T1219 Remote Access Software

Actions on Objectives (Cyber Kill Chain) ↔ Impact/Exfiltration (ATT&CK):

Finally, the attacker’s end goal (like stealing data or causing damage) aligns with ATT&CK’s Exfiltration or Impact tactics.

ID Technique
T1486 Data Encrypted for Impact
T1041 Data Exfiltration Over C2 Channel

Key Differences and Use Cases

Granularity: The Cyber Kill Chain is broader, offering a high-level view, while MITRE ATT&CK provides specific techniques for defenders to monitor, detect, and respond to.

Actionability: The Cyber Kill Chain is great for understanding the flow of an attack from start to finish, but MITRE ATT&CK offers more practical steps for defenders to take against each technique. It’s like comparing a high-level blueprint (Cyber Kill Chain) to detailed engineering plans (MITRE ATT&CK).

For example, in the Command and Control stage, ATT&CK details numerous ways an attacker might communicate with a compromised system (e.g., using DNS tunneling or encrypted communication), while the Cyber Kill Chain just categorizes it as "Command and Control."

How to Use Both in Cyber Defense

You can think of the Cyber Kill Chain as a foundation—helping to understand and structure your incident response or threat-hunting process. On the other hand, MITRE ATT&CK provides the fine details for each step, so you can improve detection and prevention at each stage.

For instance:

During the Reconnaissance stage, you could use MITRE ATT&CK to focus on detecting phishing attempts (T1566), scanning activity (T1595), or unusual domain requests.
In the Command and Control phase, using MITRE ATT&CK techniques, you could deploy tools to detect abnormal traffic patterns indicative of tunneling (T1572) or C2 beacons.
